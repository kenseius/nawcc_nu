Semantic Code is Meaningful Code

Semantics in web design refers to "the clarity of the code"; using HTML elements to convey meaning.

In other words, aligning the meaning of the content with the code.

When coding or designing, the decisions about which elements to use aren't arbitrary. Like following a map, reviewing the content grants an understanding of each element's core purpose, or role. These roles tell us both what to design and what HTML5 elements to use.

For example:

BAD SEMANTICS
---------------------
<div class="article">
<div class="title">Header Title</div>
<div class="content">Content</div>
</div>
GOOD SEMANTICS
---------------------
<article>
<h1>Header Title</h1>
<p>Content</p>
</article>

h1 gives meaning to "Header Title", whereas div.title does not.

p gives meaning to "Content", whereas div.content does not.

The idea is that through semantic markup, screen readers and browsers have a description of whatever (otherwise unspecified) content an HTML element contains. Screen readers and browsers display content based on what meaning is conveyed through the markup... It wouldn't be very helpful if every single item on a page is delivered exactly the same because they all live in meaningless div's.

Why Is Non-Semantic Markup Bad?

While both semantic and non-semantic code will visually appear exactly the same (assuming identical css styling), conveying meaning through the code is important for:

  • screen readers
  • search engines
  • modularity - so long as the html structure follows the content, and the styles follow the html structure, each individual module will remain robust, self-contained and reusable as possible.
  • maintainability - because meaning is tied to the code structure, a sense of organization is inherent; thus, the code becomes easier to scan, edit, and write.
COMMON HTML5 ELEMENTS
---------------------
