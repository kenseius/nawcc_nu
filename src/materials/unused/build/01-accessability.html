Markup

"The most basic a11y (accessibility) win is using proper elements and ARIA roles where necessary." a11y For The Masses

This is our job as the authors of the code.

This is where semantics come into play. Label content with the correct html tags, and you'll automatically remove most accessability issues.

While it may seem daunting to correctly markup every single line of code, it's a relatively simple task on its own. Therefore, code with accessability and the meaning of the content in mind right from the start, for every element you code --- this upfront investment of time pays off hugely in the middle and end of a project (plus it's especially helpful for passing an ADA scan).

The way to cut down on dev time is a modular approach --- use the available components as a template, which have much of the initial accessability best practices already added to their markup.

Vocabulary

"A big part of accessibility is language choice. Writing in a clear, concise tone." a11y For The Masses

Use clear, concise vocabulary.

This will make it easier for translators, in addition to improving the general clarity and readability of the text.
